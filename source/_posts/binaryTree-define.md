## 定义

你每天都在和"树"打交道——打开文件管理器，一层层文件夹展开下去，每个文件夹下又可以有子文件夹，这就是一棵树。文件夹没有"两个子节点封顶"的限制，但如果给每个节点加上"最多两个分支"的约束，它就成了我们今天的主角：**二叉树**。

> 二叉树(Binary Tree) 是 n (n ≥ 0)个结点的有限集合，该集合或者为空集(称为空二叉树), 或者由一个根结点和两棵互不相交的、分别称为根结点的左子树和右子树的二叉树组成。
>
> ——程杰. 《大话数据结构》

这个定义有三个关键信息：第一，空树也算二叉树；第二，左子树和右子树是**互不相交**的，不能共享节点；第三，注意定义中的递归味道——"左子树和右子树"本身又是二叉树，这意味着你可以用同样的规则一层层套下去。这种自相似的结构，是二叉树一切操作的基础。

## 特殊二叉树

不是所有的二叉树都"规整"。有些形态比较极端，有些则非常优美。下面三种是基础中的基础，后面讲堆、哈夫曼编码、二叉排序树时你会反复遇到它们。

### 斜树

> 所有的节点都只有左子树的二叉树叫左斜树。所有节点都是只有右子树的二叉树叫右斜树。这两者通称为斜树。

说白了，斜树就是"退化"成了一条线——每个节点要么只往左指，要么只往右指。左斜树看起来就像一条从左上到左下的斜线，右斜树则反过来。你会发现，**斜树本质上就是一个单向链表**，这也解释了为什么二叉树的某些操作在最坏情况下会退化成 O(n)。

### 满二叉树

> 在一棵二叉树中，如果所有分支节点都存在左子树和右子树，并且所有叶子都在同一层上，这样的二叉树称为满二叉树。

想象一座完美的金字塔：从塔尖开始，每往下一层节点数翻倍，所有"末端"整齐地落在同一水平线上——没有缺口，没有提前终结。这就是满二叉树。它是"最胖"的二叉树形态，深度为 k 时节点数恰好达到上限 2<sup>k+1</sup> - 1。

### 完全二叉树

> 对一棵具有 n 个节点的二叉树按层序编号，如果编号为 i (0 ≤ i ≤ n) 的节点与同样深度的满二叉树中编号为 i 的节点在二叉树中位置完全相同，则这棵二叉树称为完全二叉树。

满二叉树和完全二叉树经常被搞混，一个简单的区分方法是：**满二叉树是完全二叉树的特例**——满二叉树必须是"全满"的；而完全二叉树允许最后一层有缺失，但必须"从左到右连续填充"，不能跳格。

打个比方：电影院每排有 8 个座位。满二叉树 = 每排都坐满；完全二叉树 = 前面排全满，最后一排可能没坐满，但从最左开始连续坐人，中间不留空位。如果有空位在左侧、人坐在右侧，那就不是完全二叉树。

完全二叉树的重要性怎么强调都不为过——**堆结构（大顶堆、小顶堆）正是基于完全二叉树实现的**，而且因为它"紧凑"的特性，用数组存它几乎不浪费空间。

## 性质

二叉树的几个数学性质是后续算法复杂度分析的基础。别被下标吓到，每条背后都有直观的逻辑。

**性质一：第 i 层上至多有 2<sup>i-1</sup> 个节点 (i ≥ 1)。**

根节点（第 1 层）最多 1 个 = 2<sup>0</sup>，第二层最多 2 个 = 2<sup>1</sup>，第三层最多 4 个 = 2<sup>2</sup>...因为二叉树每个节点最多两个分支，每往下一层，节点数上限翻一倍，自然就是 2<sup>i-1</sup>。

**性质二：深度为 k 的二叉树至多有 2<sup>k+1</sup> - 1 个节点（根节点深度 k<sub>0</sub> = 0, k ≥ 0）。**

这是性质一的求和结果。把每一层的上限加起来：2<sup>0</sup> + 2<sup>1</sup> + ... + 2<sup>k</sup> = 2<sup>k+1</sup> - 1。一棵深度为 0 的树只有根节点，上限是 1 = 2<sup>1</sup> - 1，吻合。

> **注**：很多教材定义根节点深度为 1，此时公式写作 2<sup>k</sup> - 1。本质一样，只是深度计数的起点的区别。

**性质三：对任何非空二叉树，叶子节点数 n<sub>0</sub> = 度为 2 的节点数 n<sub>2</sub> + 1。**

这个等式初看有点神奇——叶子和"双分支节点"之间有固定关系？其实用"出度 = 入度"可以秒懂：除了根，每个节点都有一条边从父节点指过来（入度为 1）。而度为 2 的节点贡献 2 条出边，度为 1 的贡献 1 条出边。设总边数为 E，则 E = n<sub>0</sub> × 0 + n<sub>1</sub> × 1 + n<sub>2</sub> × 2 = n<sub>1</sub> + 2n<sub>2</sub>。同时 E = 总节点数 - 1（根没有入边）= (n<sub>0</sub> + n<sub>1</sub> + n<sub>2</sub>) - 1。两式联立消去 n<sub>1</sub>，得 n<sub>0</sub> = n<sub>2</sub> + 1。

**性质四：具有 n 个节点的完全二叉树的深度为 ⌊log<sub>2</sub>n⌋ + 1（⌊x⌋ 表示不大于 x 的最大整数）。**

还是用直觉理解：完全二叉树的深度取决于"最后一排排到哪"。n 个节点，按层填，填满 k 层需要 2<sup>k+1</sup> - 1 个节点，所以 n 落在 (2<sup>k</sup> - 1, 2<sup>k+1</sup> - 1] 区间内，深度自然就是 k + 1 = ⌊log<sub>2</sub>n⌋ + 1。

**性质五：完全二叉树按层序编号（从 0 开始），对任意节点 i：**

这组公式是**数组存储二叉树**的理论基础——当你把二叉树"拍平"放进数组时，靠它们就能在父子间跳转，不需要指针。

1. 若 i = 0，节点 i 是根，无双亲；否则其双亲为节点 ⌊(i - 1) / 2⌋
2. 若 2i + 1 > n，则节点 i 无左孩子；否则其左孩子为节点 2i + 1
3. 若 2i + 2 > n，则节点 i 无右孩子；否则其右孩子为节点 2i + 2

举个例子：节点 1（根节点的左孩子），其左孩子为 2×1 + 1 = 3，右孩子为 2×1 + 2 = 4；节点 1 的双亲为 ⌊(1-1)/2⌋ = 0（根），完全正确。

## 存储结构

有了上面这些定义和性质，一个自然的问题是：二叉树在计算机里怎么存？主要有两种方式——数组和链表，各有优劣。

### 顺序存储

用数组存二叉树，本质上是把性质五的层序编号当作数组下标。对于**满二叉树和完全二叉树**，节点是紧凑排列的，数组一格不浪费，而且父子关系直接用下标计算，访问极快（前序遍历时内存访问局部性尤其好）。

![](https://kinsiy-blog-img.oss-ap-southeast-1.aliyuncs.com/img/Binary_tree_in_array.svg)

但缺点是：遇到**稀疏**的树（比如斜树），就灾难了。一棵深度为 h、每个节点都只有右孩子的二叉树，实际只有 h 个节点，却需要分配 2<sup>h</sup> - 1 大小的数组——绝大多数位置都是空的。这是顺序存储的硬伤。

### 二叉链表

既然二叉树每个节点最多只有两个孩子，最自然的想法就是：每个节点存一个数据域 + 两个指针（左孩子、右孩子）。这就是二叉链表——由二叉树的"二"直接来。

```javascript
class BitNode {
  constructor(value) {
    this.value = value;
    this.left = null;
    this.right = null;
    this.parent = null; // 按需添加
  }
}
```

这个结构极其灵活，不会浪费空间（有几个节点就分配几个），插入删除只需改动指针，是二叉树最通用的存储方式。

如果需要从子节点回溯到父节点，可以在节点中增加一个指向双亲的指针域，这样就变成了**三叉链表**。
