函数声明

函数类型表达式

函数类型表达式是Ts中最简单的声明函数的方法.它看起来和ES6中的箭头函数很是相像.

(a: string) => void的意思是, 一个拥有一个string类型的参数a,不返回任何值的函数.

我们知道在Js中函数是一个对象, 可以像下面这样往函数中添加属性. (当然我们很少这么做,只是使用Function原有的属性name,length等)

这意味着我们需要声明的是一个可以调用的对象.这时候函数表达式的类型声明方法无法满足我们的要求,需要用到调用签名这一声明方法,这种方式声明也很简单,只需把函数类型表达式中的箭头 =>换成冒号 : 即可

构造签名

在Js中,还可以使用new操作符调用函数.这时候我们可以在调用签名前添加一个new,便可以声明构造函数的目的.对于调用时new操作符是可选的函数(对象),比如 Date, 需要同时声明调用签名,与构造签名

泛型函数

通常我们会写一些,输入类型与输出类型相关或者两个参数之间存在某种关联的函数.举个栗子,返回任意数组中的第一个元素

函数很简单,我们希望函数能够根据我们输入的类型自动得出返回值的类型,当我输入string[]类型时,返回值就是string类型的,其他类型同理.但实际上由于是任意数组,无法事先知道参数的类型,目前为止只能使用any[]类型,这时候返回类型自然也就是any类型.不满足我们的要求! 这就轮到泛型函数登场了

可以看到,我们在函数名后方声明了类型Type,并在参数中使用Type[]对arr进行注解,函数返回类型也是Type类型.这样我们就得到了一个通用的类型函数(泛型函数).函数的返回类型会根据参数类型自动推断出来.

注意: 不需要主动去指定Type.Ts会自动推断出正确的类型

泛型约束

当函数需要处理不同类型的对象时,在函数中我们可能只需要读/写特定的属性.只要某对象存在这些属性,就应该可以成为函数的合法参数.举例来说, 比较两个参数的length: number属性, 只要参数存在length: number属性即可比较,无其他约束.

另外对于返回泛型Type的函数, 返回值满足约束条件并不一定能满足类型Type, Type是由入参决定的,Type是“约束”的超集.

指定类型

对于泛型函数,若指定多个入参均为泛型Type,这些入参的类型应完全一样.对于不一样的类型,可以在调用函数时指定通用泛型

最佳实践

  1. 尽可能的使用类型参数本身,而不是“约束”拓展

    When possible, use the type parameter itself rather than constraining it

    Ts必须在函数调用之前推断出它的返回类型,而不是在函数调用的时候动态决定.

  2. 尽可能的少用泛型

    Always use as few type parameters as possible

    对比filter1与filter2,不难发现两种写法,在不指定类型其实是没差别的,但当指定类型时,filter2却需要多指定一个泛型.多此一举

  3. 泛型Type至少应该出现两次

    If a type parameter only appears in one location, strongly reconsider if you actually need it

    对于只出现一次的类型,完全没有必要使用泛型.记住,泛型用于关联多个值的类型

可选参数

在参数前使用?修饰,可以使参数变为可选参数.对于未指定默认值的可选参数,其类型永远都是声明类型本身与undefined的联合类型.因为在Js中未赋值的参数的值就是undefined.

与ES6一样,可以对参数设定默认值,设定默认值后,改参数变为可选参数,但不传该值或传入的值为undefined时,参数获得改默认值,并且值的类型为其本身类型,不会存在undefined

回调函数

注意,尽量不要在回调函数中使用可选参数.除非你打算在不传递参数的情况下进行回调. 具有较少参数(相同类型)的函数总是可以代替具有更多参数的函数.所以在回调中定义多个参数即可,传入回调函数定义的参数可以比定义的函数定义的参数的参数少

函数重载

针对Js中同名函数可以传入不同数量/类型的用法,除了可选/默认参数外.Ts中还可以使用函数重载实现

注意

需要区分重载函数和实现函数, 重载函数指没有函数体的函数声明,实现函数则指拥有函数体的函数.实现函数要求写在所有重载函数后,并且实现函数是兼容所有重载函数的.但是实现函数对外部不可见,这里外部指不能直接调用实现函数,只能通过重载函数签名对函数进行调用.举例来说

尽可能使用联合类型而不是函数重载.

Always prefer parameters with union types instead of overloads when possible

举例来说

对于重载,我个人理解,在Js中与可选参数差别不大,重载能做到的可选参数一样可以做到.对于不同类型,Ts中则是建议使用联合类型.

那么在实践中就是尽量不要使用重载.

  • 对相同数量参数的函数,尽量使用联合类型

  • 对不同数量参数的函数,用可选参数代替

  • 对参数数量/类型均不同的函数,结合联合类型与可选参数实现.

Void

对于拥有类型为返回void的函数类型的函数,Ts不会禁止该函数返回值,但返回值赋值给变量时,变量获取到的类型始终是void.举例来说

但当我们声明一个函数拥有void返回类型时,则禁止返回任何值
